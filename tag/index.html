Usage

Hello World shows how to add custom text to a custom tag. (View Demo) 

<!-- hello-world.html -->
<hello-world></hello-world>

<script type="module">
  // import the tag library
  import tag from './tag.js'

  // create a new tag called hello-world
  const element = tag('hello-world')

  // set the contents of the tag
  element.html(() => `Hello World`)
</script>

Advanced Usage

The Hello Name example shows the fundamentals of incorporating variables into the tag's state. (View Demo) 

<!-- hello-name.html -->
<hello-name></hello-name>

<script type="module">
  // import the tag library
  import tag from './tag.js'

  // create a new tag called hello-name
  // and set the initial state
  const { html, get } = tag('hello-name', {
    'name': 'Earth'
  })

  html(() => {
    // get the name from current state
    const { name } = get()

    // interpolate the name variable into tag content
    return `Hello ${name}`
  })
</script>

Documentation

tag(string [, object]) : object

The tag function requires a string to be used as a custom HTML tag. An optional second argument may be passed for seeding the initial state. It will return an object with the following namespaced functions.

html(function) : string | falsey
Expects a function that returns HTML as a string to be rendered into the tag. A falsey return will leave the node untouched.
css(string)
Expects a CSS stylesheet as a string. The ampersand (&) symbol may be used to scope rulesets.
on(string, string, function) : function
An event delegator that expects three arguments. An event type, a css selector, and a handler. Returns a function to remove the listener when invoked.
get() : object
Returns the current state.
set(object [, function])
Updates the current state. An optional second argument for managing the state update may be provided. That function will be called with the current state and the payload provided as the required first argument.
