Documentation: Ion Core API

The core API is five functions: on, get, set, load, relay.

Event Delegation

on(Event, Selector, Event Handler) : function

Listens for an event on the document. During that event, if the target matches the selector, the event handler is called during the capture phase.

State Management

get(String) : Object | undefined

Looks into state and returns if the provided string is a key on the state object.

set(String, Object, [function])

Merges the provided object with the portion of the state stored in the provided string. An optional function may be provided to manually merge the existing state with the new data.

The potion of the state is also backed up into IndexedDB with the string key as the identifier.

relay(function)

The provided callback function will be invoked whenever state changes and will be passed the latest state as the sole argument.

load(String)

Recovers a portion of state from IndexedDB. Any relays will be notified after the recovered state is ready.
