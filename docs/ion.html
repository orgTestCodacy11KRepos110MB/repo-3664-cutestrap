Documentation: Ion

Ion was written using modern web technologies. The core of the library was written as a portable, reusable web application called Ion.

The rationale for writing fresh code is two-fold:

  • Build a lightweight solution to the two most common challenges in web development: Event Delegation and State Management.
    • (We then crank these up to Eleven.)
  • Do it with zero dependencies outside of a modern web browser.

Browser Support: the most recent feature utilized is JavaScript Modules. This leaves our support tail at Firefox 60, released 2018-05-09.

Event Delegation

Our event delegation is a thin wrapper around addEventListener and matches. By combining the two, we get a single function call for delegating events to elements.

Modern web applications are expected to dynamically update content based on state changes. We've added a custom 'render' event to notify elements when the state changes.

We also utilize a MutationObserver under the hood. Using the same 'render' event, an element is also notified when itself is updated. This enables our HTML to become self-aware. Self-aware HTML is able to make external network requests based on the document. (And that's just one example!)

State Management

Our state management is backed by IndexedDB. A string key is used to identify a piece of data within the storage mechanism.

Changes to state will notify any callback functions that have been provided. This is how we notify elements that are listening for our 'render' event.
